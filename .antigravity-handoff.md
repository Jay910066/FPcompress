## 3. 問題根因分析 (Root Cause Analysis)
* **Speed GPU 的 Shared Memory 越界錯誤 (Out-Of-Bounds Bug)**:
    * 發生在 `speed-compressor` 與 `speed-decompressor` 的 Double/Single 精度版本 (共 4 個檔案)。
    * 核心原因：共用記憶體 `chunk` 陣列宣告大小不足以覆蓋 Kernel 內對 `chunk[last]` 的存取，並且 `block_sum_reduction` 需要額外的暫存空間，導致記憶體越界寫入與讀取 (Illegal memory access)。
    * 解決方案：需將陣列大小統一提升至 `2 * (CS / sizeof(long long)) + WS + 1` (4129 元素)。
* **DPratio GPU 的 memset 初始化錯誤 (Partial Memset Bug)**:
    * 發生在 `double_src/ratio-compressor-double.cu` 第 351 行。
    * 核心原因：`d_fullcarry` 陣列型態為 `long long*`，但在呼叫 `cudaMemset` 時，清空的記憶體大小被寫死為 `chunks * sizeof(byte)`。這導致只有 1/8 的陣列被歸零，殘留的垃圾值破壞了 Carry 傳遞的邏輯，導致壓縮資料流損毀。
    * 解決方案：將 `sizeof(byte)` 修正為 `sizeof(long long)`。
    * 註：我們已檢查過 `single_src/ratio-compressor-single.cu`，其陣列為 `int*` 且配置為 `chunks * sizeof(int)`，是正確的，不需修改。
